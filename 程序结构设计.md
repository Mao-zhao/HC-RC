## 3. 数据流设计

### 3.1 整体数据流

```
EuRoC 原始数据
    │
    ▼
[data/dataset.py]        ← 加载 .csv，滑动窗口采样
    │
    ├──→ train_loader    (8911 samples)
    ├──→ val_loader      (1957 samples)
    └──→ test_loader     (2182 samples)
            │
            ▼
[models/*.py]            ← 加载拓扑，构建模型
    │
    ▼
[training/*.py]          ← 训练 + 评估
    │
    ├──→ results/exp*/    ← 模型权重 + 评估指标 + 预测结果
    │
    ▼
[visualization/*.py]     ← 读取结果 + 数据，生成图表
    │
    ▼
figures/                 ← 最终图表
```

### 3.2 数据格式约定

**输入张量**：`(batch, seq_len, 6)` — 6 维 = [x, y, z, vx, vy, vz]

**输出张量**：`(batch, 10, 3)` — 10 步预测的 3D 位置

**训练数据文件**：
- `train_data.npy`：`(8911, 30, 6)` — 观测窗口
- `train_target.npy`：`(8911, 10, 3)` — 预测目标
- 同理 `val_data.npy`, `val_target.npy`, `test_data.npy`, `test_target.npy`

**拓扑文件**（预计算，避免重复构建）：
- `topology/cx_adjacency.npy`：`(25, 25)` — CX 邻接矩阵
- `topology/cx_nonbacktracking.npy`：`(322, 322)` — 非回溯矩阵
- `topology/cx_heterogeneous_mask.npy`：`(3422, 6)` — HBM 掩码

### 4.11 models/reservoir.py

**功能**：储池计算基类（PyTorch Module）。

```python
class ReservoirBase(nn.Module):
    """
    Leaky-Integrator Echo State Network 基类
    
    核心公式:
        h_tilde = tanh(W_in @ x_t + W_res @ h_{t-1} + W_fb @ delta_p_{t-1} + b)
        h_t = (1 - gamma) * h_{t-1} + gamma * h_tilde
        delta_p_t = W_out @ h_t
        p_{t+1} = p_t + delta_p_t
    """
    
    def __init__(self, N, input_dim=6, output_dim=3, gamma=0.8):
        self.W_in = nn.Parameter(...)   # 冻结
        self.W_res = nn.Parameter(...)  # 冻结
        self.W_fb = nn.Parameter(...)   # 冻结
        self.W_out = nn.Linear(N, output_dim)  # 唯一可训练
        self.gamma = gamma
    
    def forward(self, x_obs, targets=None, teacher_forcing=True):
        """
        Args:
            x_obs: (batch, 30, 6) 观测序列
            targets: (batch, 10, 3) 目标位置（训练时用于 teacher forcing）
            teacher_forcing: 是否使用 teacher forcing
        
        Returns:
            predictions: (batch, 10, 3) 预测轨迹
        """
```

**关键设计**：
- `W_in`, `W_res`, `W_fb` 设置为 `requires_grad=False`，确保零梯度
- 仅 `W_out` 可训练
- `forward` 内实现自回归预测循环

### 4.17 training/evaluate.py

```python
def compute_ade(predictions, targets):
    """Average Displacement Error"""
    return torch.mean(torch.norm(predictions - targets, dim=-1)).item()

def compute_fde(predictions, targets):
    """Final Displacement Error"""
    return torch.mean(torch.norm(predictions[:, -1] - targets[:, -1], dim=-1)).item()

def compute_stepwise_error(predictions, targets):
    """逐步累积误差（每步的 ADE），返回 shape (10,)"""
    return torch.mean(torch.norm(predictions - targets, dim=-1), dim=0).cpu().numpy()

def compute_all_metrics(predictions, targets):
    """返回 ADE, FDE, stepwise_error 的字典"""
    return {
        "ADE": compute_ade(predictions, targets),
        "FDE": compute_fde(predictions, targets),
        "stepwise_error": compute_stepwise_error(predictions, targets),
    }
```

```python
    - CX-Direct-RC (78 params)
    
    输出：
    - results/exp1/{model_name}/metrics.json
    - results/exp1/{model_name}/predictions.npy
    - results/exp1/{model_name}/model.pt
    """
    models = [
        ("residual_lstm", ResidualLSTM(), RNN_TRAINING),
        ("tiny_rnn", TinyResidualRNN(), RNN_TRAINING),
        ("residual_rnn", ResidualRNN(), RNN_TRAINING),
        ("cx_direct_rc", CXDirectRC(), RC_TRAINING),
    ]
    
    for name, model, config in models:
        if is_rc_model(model):
            trainer = RCTrainer(model, config)
        else:
            trainer = RNNTrainer(model, config)
        history = trainer.train(train_loader, val_loader, epochs=100)
        metrics = trainer.evaluate(test_loader)
        logger.save_results(name, metrics)
```

## 附录：与现有代码的关系

为避免重复工作，本项目复用以下已有代码中的核心逻辑：

| 现有代码 | 本项目对应模块 | 复用方式 |
|---------|-------------|---------|
| `true_cx_reservoir.py` | `models/rc_*.py` | 抽取储池构建逻辑，模块化 |
| `N-Aligned Controls.py` | `experiments/exp2_topology_ablation.py` | 控制实验逻辑 |
| `Heterogeneous_CX_reservoir.py` | `experiments/exp3_heterogeneity.py` | 异质掩码逻辑 |
| `train_baseline_RNNmodels.py` | `models/rnn_residual.py`, `models/lstm_residual.py` | RNN/LSTM 架构 |
| `真实CX储池计算_数学模型.md` | `数学模型.md` | 数学公式和理论依据 |

**设计原则**：
1. 保持与现有代码一致的残差 Delta 预测范式
2. 统一使用 PyTorch 框架
3. 模块化拆分，使每个文件职责单一
4. 配置与代码分离，方便调参
5. 所有结果保存为标准化格式，便于可视化读取
